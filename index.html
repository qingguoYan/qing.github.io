eclipse访问不到tomcat原因

Posted on 2018-05-21 
症状：

tomcat在eclipse里面能正常启动，而在浏览器中访问http://localhost:8080/不能访问，且报404错误。同时其他项目页面也不能访问。

关闭eclipse里面的tomcat，在tomcat安装目录下双击startup.bat手动启动tomcat服务器。访问htt://localhost:8080/能正常访问tomcat管理页面。

症状原因：

eclipse将tomcat的项目发布目录（tomcat 目录中的webapp）重定向了,所以你会发现在tomcat安装目录下的webapp目录里面找不到你的项目文件。

解决办法：

重新配置下tomcat服务器：

在eclipse中的server页面，双击tomcat服务，会看到如图所示的配置页面：

可以看到红圈中选择的是 Use workspace metadata(does not modify Tomcat installion)

如果该tomcat中部署了项目的话，这红圈中的选项会灰掉不能修改，要修改必须得先把tomcat中的部署的服务都移除。

如图：

通过右键单击tomcat服务器选择 Add and Remove，在弹出的对话框中移除已部署的项目。移除完确定后，将看到上面的选项面板部分可编辑了。

选择Use tomcat installation(Task control of Tomcat installation) 即选择tomcat的安装目录来作为项目的发布目录。

然后,下来四行,看到”Deploy Path”了没?它后面的值默认是”wtpwebapps”,把它改成”webapps”,也就是 tomcat

中发布项目所在的文件夹名字。
修改后关掉该页面，保存配置。这样就将项目部署到了tomcat安装目录下的webapp
重启tomcat服务器，访问http://localhost:8080则能正常访问了，自己部署的项目也能正常访问了。

搭建属于自己的博客

Posted on 2018-05-19 
总体步骤:
           一.环境准备
           1.下载git
           2.下载NodeJs
           3.hexo--------执行以下命令完成hexo的安装

              3.1 npm install hexo-cli -g
              3.2 npm install hexo-deployer-git
              3.3 npm install hexo-generator-feed
              3.4 npm install hexo-generator-index
              3.5 npm install hexo-generator-archive
              3.6 npm install hexo-generator-tag
              3.7 npm install hexo-generator-category
              3.8 npm install hexo-generator-sitemap
              3.9 npm install hexo-generator -baidu-sitemap
            二.初始化博客

            1.创建一个文件夹myblog,并cd到此文件夹
            2.执行命令hexo init
            3.npm install
            三.配置博客

            打开根目录下_config.yml文件
            可以参考网上数据进行修改
            注意:每一个冒号后面都要有一个空格，否则配置不会起作用
            更换主题需要更改theme参数配置

            四.创建博文
            hexo new "（你的博文名字）"
            hexo s 启动hexo服务
            浏览器输入提示的网址即可进入博客

            五关联github
            如果想把博文同步到GitHub上,首先申请一个github账号，然后添加sshKey
            在git上执行 ssh-keygen -t -rsa -C "你的邮箱"

            复制你的密匙在github的Setting中添加你的sshKeys

            最后新建一个Repository,命名为 qingxiaoYan.github.io

            最最后在_config.yml文件里配置deploy信息
             deploy:
                 type: git
                 repository: git@github.com: "qingxiaoYan/qing.github.io.git"
                 branch: master



            六.执行 hexo clean
                    hexo g
                    hexo d

               ok！！！

三种排序算法

Posted on 2018-05-19 
                                 冒泡排序算法
  关键代码如下:

for(int i=0;i<arr.length;i++){
    for(int j=0;j<arr.length-1-i;j++){
        if(arr[j]>arr[j+1]){
          int temp=arr[j];
          arr[j]=arr[j+1];
          arr[j+1]=temp;
        }
    }
}

                                选择排序算法
  关键代码如下：

for(int i=0;i<arr.length;i++){
      for(int j=i+1;j<arr.length;j++){
            if(arr[i]>arr[j]){
                int temp=arr[i];
                arr[i]=arr[j];
                arr[j]=temp;
            }
      }
}         


                                直接插入算法

  关键代码如下:

 for(int i=1;i<arr.length;i++){

      int j=i-1; //无序的坐标
      temp=arr[i];//保存即将插入的元素
      while(j>=0 && temp<arr[j]){ 

          arr[j+1]=arr[j];;//如果满足条件，有序序列往后移动一位
          j--;
      }

      arr[j+1]=temp;//在插入的位置插入元素

 }
